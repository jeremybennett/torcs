For Linux From "all-in-one" Source Package

  1. Check the dependencies
  2. Download the source package torcs-1.2.4.tar.bz2.
  3. Unpack the package with "tar xfvj torcs-1.2.4.tar.bz2".
  4. Run the following commands: 

    $ cd torcs-1.2.4
$ ./configure        # --prefix="target dir", --enable-debug or --disable-xrandr might be of interest
$ make
$ make install
$ make datainstall

    Default installation directories:

    • /usr/local/bin - TORCS command (directory should be in your PATH)
    • /usr/local/lib/torcs - TORCS dynamic libs (directory MUST be in your LD_LIBRARY_PATH if you don't use the torcs shell)
    • /usr/local/share/games/torcs - TORCS data files


  5. Run the "torcs" command (default location is /usr/local/bin/torcs), you can use those command line options.
    All the configuration data, race results and players options will be saved below the $HOME/.torcs directory.

For Windows From "all-in-one" Source Package

  1. Requires VC++ 6.0 (tested with sp6).
  2. Download the source package torcs-1.2.4.tar.bz2.
  3. Unpack the package with "tar xfvj torcs-1.2.4.tar.bz2" or any application you like, be careful to select a path without whitespaces and other special characters.
  4. Hint: You can build a release and a debug version side by side, the required step for the debug version is showed in brackets "()".
  5. Open a command prompt and cd into the torcs-1.2.4 directory.
  6. Run setup_win32.bat (setup_win32_debug.bat).
  7. Run setup_win32-data-from-CVS.bat (setup_win32-data-from-CVS_debug.bat).
  8. Open the file TORCS.dsw with vc++ 6.0.
  9. Select the TORCS project and the win32-Release (win32-Debug) version.
  10. Compile the project (release 0 warnings, debug 131 irrelevant warnings about too long debug symbols).
  11. cd into the "runtime" ("runtimed") directory.
  12. Run "wtorcs.exe".

For Linux From Separated Source Packages

  1. Check the dependencies
  2. Download the following source packages:
    • TORCS-1.2.4-src.tgz (mandatory)
    • TORCS-1.2.4-src-robots-base.tgz (mandatory)
    and optionally additional opponents, of course you should get all:
    • TORCS-1.2.4-src-robots-berniw.tgz
    • TORCS-1.2.4-src-robots-bt.tgz
    • TORCS-1.2.4-src-robots-olethros.tgz

  3. Unpack the packages (tar xvfz filename.tgz)
  4. Run the following commands: 

    $ cd torcs-1.2.4
$ ./configure        # --prefix="target dir", --enable-debug or --disable-xrandr might be of interest
$ make
$ make install

    Default installation directories:

    • /usr/local/bin - TORCS command (directory should be in your PATH)
    • /usr/local/lib/torcs - TORCS dynamic libs (directory MUST be in your LD_LIBRARY_PATH if you don't use the torcs shell)
    • /usr/local/share/games/torcs - TORCS data files


  5. Download the following data packages:
    • TORCS-1.2.4-data.tgz (mandatory, system data)
    • TORCS-1.2.4-data-tracks-road.tgz (mandatory, road-circuit tracks)
    • TORCS-1.2.4-data-cars-extra.tgz (mandatory, cars)
    • TORCS-1.2.4-data-tracks-oval.tgz (oval-like tracks, optional)
    • TORCS-1.2.4-data-tracks-dirt.tgz (dirt tracks, optional)
    • TORCS-1.2.4-data-cars-nascar.tgz (Nascar cars, optional)
    • TORCS-1.2.4-data-cars-Patwo-Design.tgz (rally cars, optional)
    • TORCS-1.2.4-data-cars-kcendra-gt.tgz (GT cars from the 60's, optional)
    • TORCS-1.2.4-data-cars-kcendra-roadsters.tgz (roadsters from the 60's, optional)
    • TORCS-1.2.4-data-cars-kcendra-sport.tgz (pport cars from the 60's, optional)
    • TORCS-1.2.4-data-cars-VM.tgz (race cars, optional)

  6. Go to the /usr/local/share/games/torcs or "target dir"/share/games/torcs directory.
  7. Untar the data packages.
  8. Run the "torcs" command (default location is /usr/local/bin/torcs), you can use those command line options.
    All the configuration data, race results and players options will be saved below the $HOME/.torcs directory.
For more detailed installation instructions have a look at Bernhard "Berniw" Wymann's site.

Linux From Sources CVS

You'll need those dependencies

Getting the sources

You have to run the following commands to get the sources (might somtimes not compile or crash):

cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/torcs login

The main trunk is:

cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/torcs co -P torcs

Get more info on CVS

Dependencies

Requisites for Linux

You will need:
  • Hardware accelerated OpenGL (usually provided by your distro).
  • GLUT 3.7.
  • Or FreeGlut (better for full screen support than GLUT).
  • PLIB 1.8.3 version.
  • Modified OpenAL for optimal sound.
  • libpng and zlib (usually provided by your distro).

TORCS Command Line Options

  • -l list the dynamically linked libraries
  • -d run under gdb and print stack trace on exit, makes most sense when compile with --enable-debug
  • -e display the commands to issue when you want to run under gdb
  • -s disable multitexturing, imortant for older graphics cards
  • -m use X mouse cursor and do not hide it during races

TORCS for your Linux distribution

TORCS has been adopted from a lot of distributions (e.g. SuSE, Debian, Fedora, Mandriva, Gentoo, ...), have a look at you installation CD's/DVD's or online repositories. Be aware that some of these versions have been stripped down to reduce the package size.

Experimental Linux Binary with Installer

This binary is dynamically linked and will probably not run on your system, just try it if you have bandwith and time to waste. Please share your experiences with us. You can get it here. Install it with "sh torcs-1.2.4-linux-i686-experimental.bz2.run". To resolve missing dynamic libraries you can run "torcs -l" (have a look at the command line options). It requires at least an i686 compatible CPU to run.

Binary Installer Package For Windows

  • Download and run torcs_1_2_4_setup.exe. It requires at least an i686 compatible CPU to run.
